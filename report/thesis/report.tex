\begin{abstract}
% Moti
In the HPC world, the communication of massive parallel workloads in distributed memory systems thoroughly is analysed.
GPU hardware and software continues to scale in parallelism, yet the intra-application communication hasn't been  a topic of research. However, the understanding of this intra-application communication is of fundamental importance for multi-GPU scaling of applications.

This work uses compiler extensions to instrument GPU applications for tracing. The instrumentation adds tracing for all load and store instructions targeting global memory in a GPU. Data is dynamically transferred from the device to host at run time, through a producer-consumer controlled buffer. 

Using the generated traces, communication across BSP supersteps is analysed. The analysis shows messages between supersteps are dominated by few transfer sizes. The number of communication partners and memory strides in communication depend on the application.
Irregular applications display irregularities in the amount of data and number of communication partners across several supersteps, while regular applications do not. All applications show a tendency towards collective communication.
\\\\
\hfill \centerline{\textbf{Zusammenfassung}}\hfill
\\\\
In der Welt des High-Performance Computing ist die Kommunikation von distributed-memory Anwendungen ausgiebig erforscht. Die Parallelität von GPU Hardware und Software wächst immer weiter, aber die Kommunikation innerhalb von Anwendungen wurde bis jetzt noch nicht untersucht. Das Verständnis dieser Kommunikation ist aber essenziell für Multi-GPU Skalierung von Anwendungen.

Diese Arbeit nutzt Compiler-Erweiterungen, um GPU Anwendungen so zu instrumentieren, dass diese Traces erzeugen. Das Tracing geschieht für alle Speicherzugriffe auf den Global Memory einer GPU. Die Daten werden dynamisch von
der GPU auf den Host übertragen, über einen Erzeuger-Verbraucher gesteuerten Speicher.

Die generierten Daten werden dazu verwendet, die Kommunikation über BSP Supersteps zu analysieren. Eine Analyse
zeigt die Übertragungen zwischen Kerneln sind von einigen wenigen Größen dominiert. Die Anzahl der Kommunikationspartner und Größe des Memory Strides während der Kommunikation hängen von der Anwendung ab. Irreguläre Anwendungen zeigen Unregelmäßigkeiten im übertragenen Datenvolumen und der Anzahl
der Kommunikationspartner über mehrere Supersteps hinweg, wohingegen reguläre Anwendungen das nicht zu tun. All Anwendungen zeigen eine Tendenz zur kollektiven Kommunikation.
\end{abstract}
